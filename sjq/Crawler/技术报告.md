# 全国公共资源交易平台（山西省）爬虫 — 技术报告

**目标网站**: `https://prec.sxzwfw.gov.cn`  
**爬取范围**: 工程建设栏目下 4 个信息类型  
**运行环境**: Linux 
**文档日期**: 2026-06-17

---

## 一、爬取目标

### 1.1 平台架构

「全国公共资源交易平台（山西省）」是山西省公共资源交易的统一入口。爬虫聚焦其 **工程建设（gc）** 子栏目，覆盖招投标完整生命周期的 4 个信息类型：

```
https://prec.sxzwfw.gov.cn
  └── 工程建设
        ├── /jyxxzbjh/    招标计划         （计划阶段）
        ├── /jyxxgczb/    招标/资审公告     （招标阶段）
        ├── /jyxxgchxr/   中标候选人公示     （评标阶段）
        └── /jyxxgcgs/    中标结果公示       （定标阶段）
```

### 1.2 信息类型关系

```
招标计划 ──→ 招标/资审公告 ──→ 中标候选人公示 ──→ 中标结果公示
 (预告)       (正式招标)        (评标公示)         (最终结果)
```

4 个类型构成时序链：**计划→招标→评审→定标**，覆盖工程建设招投标全过程。

## 二、系统架构

### 2.1 模块结构

```
├── config.py        全局配置（URL、延迟、UA 池、代理源）
├── proxy_pool.py    代理 IP 池管理（本地文件 + 免费 API 双通道）
├── crawler.py       核心爬虫（列表爬取、详情解析、反封策略）~900 行
└──  main.py          入口脚本（参数处理、结果输出、统计摘要）
```

### 2.2 爬取流程

```
┌─────────────────────────────────────────────────────┐
│  栏目初始化 → 会话预热(访问首页)                       │
│     ↓                                                │
│  列表页第1页 → 解析(标题+日期+ID+交易场所) → 日期过滤  │
│     ↓                                                │
│  翻页(第2-N页) → 重复 → 遇过期记录则停止               │
│     ↓                                                │
│  逐个详情页 → HTML解析    │
│     ↓                                                │
│  JSON + CSV 双格式输出                                │
└─────────────────────────────────────────────────────┘
```

### 2.3 技术选型

| 决策 | 理由 |
|------|------|
| 标准库解析 HTML（`re` + `html.parser.HTMLParser`） | 无 pip，无法安装 bs4/lxml |
| 仅依赖 `requests` | 系统自带唯一 HTTP 库 |
| 非 AJAX，SSR HTML 解析 | 站点为服务端渲染，无 JS 动态加载 |
| 覆盖模式输出（`"w"`） | 每次运行重新爬取全部数据 |

## 三、4 个信息类型 HTML 结构分析

### 3.1 共性结构（gczb / gchxr / gcgs）

三个信息类型共享相同的 HTML 布局：

```
<div class="cs_xq_content">
  ├── <div class="div-title2">    ← 发布时间行
  │     └── "公告发布时间：..." / "公示发布时间：..."
  │
  └── <div class="div-article2">  ← 正文区域
        └── <table class="gycq-table">
              └── <tr><td> ... 正文内容 ... </td></tr>
```

### 3.2 各类型正文结构差异

| 结构特征 | 招标计划 (zbjh) | 招标/资审公告 (gczb) | 中标候选人公示 (gchxr) | 中标结果公示 (gcgs) |
|---------|:---:|:---:|:---:|:---:|
| 正文容器 | gycq-table | div-article2 > table | div-article2 > table | div-article2 > table |
| 标题提取方式 | `<title>` 标签 | `<p style="font-size:XXpx">` | `<h1>` 或居中 `<p>` | 纯文本首行 |
| 发布时间标签 | 无 | `公告发布时间` | `公示发布时间` | `公示发布时间` |
| 字段格式 | 稀疏 `<td>` 键值对 | 段落式一、二、三、 | 段落式 + 候选人表格 | 段落式或扁平式 |
| 编号格式 | 投资项目统一代码 | `(招标编号：XXX）` | `(招标编号：XXX)` | `项目编号：XXX` |
| 发布日期 | 列表页自带 | 列表页 + 正文重复 | 列表页 + 正文重复 | 列表页 + 正文重复 |
| 文档变种 | 无 | 控制价公告、终止公告、流标公告 | 更正公告、延期公告 | 更正公告、结果公告 |

### 3.3 招标计划（zbjh）— 独立结构

```
<div class="cs_xq_content">
  └── <table class="gycq-table">
        ├── <tr><td>投资项目统一代码：</td><td>2509-140522-...</td></tr>
        ├── <tr><td>项目名称：</td><td>阳城县太岳烈士陵园...</td></tr>
        ├── <tr><td>项目类型：</td><td>房屋市政</td></tr>
        └── ...（共 16 对键值，按 TDi/i%2 稀疏配对）
```

提取策略：用 `TableFieldExtractor(HTMLParser)` 监听 `<td>` 数据事件，按奇偶位置配对。

### 3.4 HTML 标签拆散问题（全站共性）

该站大量使用内联 `<span>` 包裹单个字符或汉字，去标签后产生多余空格：

```html
<!-- HTML 原始 -->
<span>招标编号</span><span>：</span><span>SXXZY-GCG</span><span>K</span><span>-2</span><span>6</span><span>0</span><span>14</span><span>）</span>

<!-- 去标签后 -->
招标编号 ： SXXZY-GCG K -2 6 0 14 ）
```

### 3.5 括号混用问题

正文中全角 `（）` 和半角 `()` 括号混用：

```
（招标编号：SXXZY-GCGK-26014）   ← 全角括号（主流）
(招标编号：ZHLH-2026-011)       ← 半角括号（少数）
```

`crawler.py` 原正则仅匹配半角 `[)]`，导致约 55% 的编号提取失败。

## 四、字段提取方法

### 4.1 `_extract_content_text()` — 正文区域限定

限定 `cs_xq_content` → footer 标记区间提取纯文本，防止 JS/CSS 污染：

1. 定位 `cs_xq_content` 起止
2. 移除 `<script>` / `<style>` 标签
3. 移除 HTML 标签
4. 规范化空白

### 4.2 `_extract_title_from_content()` — 三级标题提取

| 优先级 | 策略 | 适用类型 |
|--------|------|---------|
| 1 | `<h1>` 标签内容 | gchxr |
| 2 | `font-size:XXpx` 居中 `<p>` | gczb |
| 3 | 正文首行有意义文本 | gcgs |

**缺陷**: 策略 2 的 font-size 正则 `1[68]px` 只覆盖 16/18px，实际有 19px/21px 等变体。

### 4.3 `_extract_from_body_text()` — 全文标签边界提取

从纯文本按字段标签匹配值，到下一个字段边界截断：

```
label_pattern: "招标人[：:]"
boundary:      "地址|电话|联系|电子邮|项目名|招标项|招标方|招标代|监督部|..."
```

### 4.4 `fix_empty_fields.py` — 辅助修复脚本

因 `crawler.py` 中部分正则无法匹配实际 HTML 变体，编写独立修复脚本从已有的全文文本字段重新提取空值字段：

| 功能 | 说明 |
|------|------|
| `extract_title()` | 4 层 fallback 提取项目名称 |
| `extract_zbbh()` | 3 层 fallback 提取招标编号/项目编号 |
| `extract_area()` | 提取招标项目所在地区 |
| `extract_gssj_start/end()` | 3 层 fallback 提取公示时间 |
| `extract_zbr()` | 提取中标人 |
| `clean_span_spaces()` | 合并 `<span>` 拆散的字母数字 |

## 五、反封策略

### 5.1 请求层

| 措施 | 参数 | 说明 |
|------|------|------|
| UA 轮换 | 7 个 Chrome/Firefox/Edge UA | 每次请求随机 |
| Referer 链 | 首页→列表页→详情页 | 模拟浏览器导航 |
| 会话预热 | 先 GET 首页 | 建立 Cookie/Session |
| 完整请求头 | Accept / Accept-Language / Cache-Control / Connection / Upgrade-Insecure-Requests | 模拟浏览器 |

### 5.2 速率控制

| 措施 | 延迟 | 说明 |
|------|------|------|
| 详情页间隔 | 5-10 秒（随机） | 模拟人工阅读 |
| 翻页间隔 | 8-15 秒（随机） | 翻页为高风险操作 |
| 栏目冷却 | 60-120 秒（随机） | 切换信息类型间隔 1-2 分钟 |
| 栏目顺序 | 随机打乱 | 避免固定爬取模式 |

### 5.3 容错与重试

| 措施 | 参数 |
|------|------|
| 最大重试 | 3 次 |
| 退避策略 | 指数退避 `3 × 2^n` 秒 + 30% 随机抖动 |
| 429 处理 | 等待 `(attempt+1) × 15` 秒 |

### 5.4 日期过滤

- 列表页按发布日期降序
- 日期比较仅对比年月日（`.date()`），避免含时分秒的 Bug
- 最多翻页 30 页（安全上限）

## 六、代理 IP 池

### 架构

```
本地文件 (proxies.txt) → 并发验证 → 有效代理入库
不足时 → 免费 API (89ip/ip3366) → 验证 → 去重 → 入库
运行时 → 失效剔除（≥3 次）→ 阈值以下自动补充
```

### 实测结论

89ip + ip3366 双源获取 60 个免费代理，100% 验证失败。免费代理生命周期极短（< 5 分钟），且服务器无法访问外网。当前环境只能直连 `--no-proxy`。
